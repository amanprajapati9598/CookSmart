# CHAPTER 2: LITERATURE REVIEW AND GAP ANALYSIS

## 2.1 Literature Review

### 2.1.1 Introduction to the Domain
The domain of Food Technology and Culinary Management has evolved significantly from traditional cookbooks to advanced digital platforms. Modern applications integrate mobile computing, data management, and artificial intelligence to assist users in cooking, nutrition tracking, and inventory management.

With increasing health awareness and time constraints in daily life, the demand for intelligent and efficient cooking solutions has grown rapidly. This industry has become a multi-billion dollar sector where systems aim to provide personalized experiences based on user preferences, dietary requirements, and available ingredients.

### 2.1.2 Review of Existing Systems / Research Papers

**Reference 1: Yummly – Smart Cooking Discovery**
- **Author:** Yummly Inc.
- **Year:** 2011–2024
- **Objective:** Personalized recipe recommendations and grocery integration
- **Technology:** Python, Native Mobile Apps
- **Methodology:** Data-driven personalization
- **Key Features:** Advanced filters, meal planner, ingredient recognition
- **Limitations:** Premium features require subscription, limited community focus

**Reference 2: Cookpad – Social Recipe Sharing**
- **Author:** Akimitsu Sano et al.
- **Year:** 2020
- **Objective:** Build global recipe-sharing community
- **Technology:** Ruby on Rails, React Native
- **Methodology:** User-generated content
- **Key Features:** Community engagement, regional recipes
- **Limitations:** No pantry tracking, inconsistent content quality

**Reference 3: BigOven – Inventory Focused System**
- **Author:** BigOven.com
- **Year:** 2023
- **Objective:** Reduce food waste using inventory tracking
- **Technology:** .NET, SQL Server
- **Methodology:** Rule-based matching
- **Key Features:** Grocery list, meal planner
- **Limitations:** Ad-heavy interface, internet dependency

**Reference 4: Tasty – Media-Based Cooking Platform**
- **Author:** Buzzfeed Inc.
- **Year:** 2024
- **Objective:** Provide visual cooking experience
- **Technology:** JavaScript, Video APIs
- **Methodology:** Video-based learning
- **Key Features:** Short videos, guided cooking
- **Limitations:** Focus on entertainment, lacks pantry system

### 2.1.3 Comparative Study Table

| System Name | Technology Used | Key Features | Limitations |
| :--- | :--- | :--- | :--- |
| **Yummly** | Python, Mobile Apps | AI recommendations | Paid features |
| **Cookpad** | Ruby on Rails | Community sharing | No inventory system |
| **BigOven** | .NET, SQL | Meal planning | Ad-heavy UI |
| **Tasty** | JS, APIs | Video tutorials | No pantry tracking |
| **CookSmart (Proposed)** | Flutter, PHP, MySQL | All-in-one system | Local server dependency |

### 2.1.4 Tools & Technologies Studied
- **Programming Languages:** Dart, PHP
- **Database:** MySQL
- **Framework:** Flutter
- **Tools:** XAMPP, Postman

### 2.1.5 Summary of Findings
- Existing systems lack integration of multiple features
- Most apps focus either on social or utility, not both
- Pantry tracking and automation are missing
- Need for offline-capable intelligent systems
- Gap exists in hands-free and AI-based cooking assistance

## 2.2 Gap Analysis

### 2.2.1 Problem Statement
Users face difficulty in finding recipes based on available ingredients, which leads to food wastage and inefficient meal planning. Existing systems are either complex, costly, or incomplete in functionality.

### 2.2.2 Limitations of Existing Systems
- Manual pantry tracking
- No real-time updates
- High subscription cost
- Poor UI/UX design
- Limited offline functionality
- Lack of integrated system

### 2.2.3 Identified Gaps

| Existing System Issue | Gap Identified | Proposed Improvement |
| :--- | :--- | :--- |
| Fragmented features | No all-in-one system | Integrated platform |
| Ad-heavy apps | Poor user experience | Clean premium UI |
| Weak inventory link | No auto updates | Smart pantry sync |
| Static recipes | No interactivity | Voice-based cooking |

### 2.2.4 Proposed System Advantages
- Automation of pantry and meal planning
- Improved data security
- Faster processing using Flutter
- Reduced human error
- Modern and user-friendly interface
- Future-ready analytics support

### 2.2.5 Justification of Project
- Provides a complete smart kitchen solution
- Aligns with modern AI and automation trends
- Technically feasible using open-source tools
- Cost-effective and scalable
- High relevance in food-tech and lifestyle industry
